How scoring works

The editor score is a simple sum. Each category is scored from 0 up to its maximum points, then we add them together to get a final score out of 100. The sections inside each server card mirror these same categories, so the rubric and the review topics match one to one.

Exclusion rules (must pass)

  • No pay-to-win: no progression advantages sold for money.
  • Clear vanilla-first statement: changes and additions must be disclosed.
  • Public rules: griefing, PvP, resets, and moderation expectations.
  • Contact exists: at least an active Discord or website.
  • Verifiable access: IP and availability can be checked (we show â€œlast checkedâ€).

Editor score (0 to 100)

  • 35 Vanilla purity
  • 20 Fairness and no P2W
  • 15 Stability and last checked
  • 15 Rules and moderation
  • 10 Community and longevity
  • 5 Transparency

Example: if a server scores 30/35 for purity and 18/20 for fairness, those points are added directly into the final score.

Vanilla Hytale server FAQ

Definitions stay simple on purpose, so â€œvanillaâ€ remains useful as a label and not just a marketing word.

What is a vanilla Hytale server? Definition

A vanilla Hytale server focuses on base gameplay and avoids changes that significantly alter progression, balance, loot, or economy. Some servers may still run minimal quality-of-life and moderation tooling, as long as it is disclosed and does not rewrite the core experience.

Vanilla vs semi-vanilla Difference

Semi-vanilla usually adds convenience systems. In this list, we prefer vanilla-first servers and we document any gameplay footprint clearly per server.

Do you accept pay-to-win servers? Fair play

No. Pay-to-win fails the exclusion rules and is not listed.

Can I suggest a server? Submissions

Yes. Send the server name, IP, region, language, and links to rules and monetization policy. If it passes the exclusion rules, we review it.
